## 4. ANALYSIS FEATURES

- **STDF Compliance Framework (C1-C10)** -- 10 dimensions with sub-criteria, each weighted (critical, high, good-to-have). Dimensions include: cost curve analysis, capability curve analysis, adoption curves, tipping points, data integrity, and others. Used for both guiding new analyses and evaluating existing ones. (Dr. Ali: "It is adding the generalizability aspect into the framework.")

- **AI/Labor (AL) Disruption Model** -- bottom-up model starting from 341 BLS occupations, broken into 12,000 tasks (from O*NET), further decomposed into 90,000 sub-tasks. For each sub-task: token estimate, cognitive classification, time estimate. Tipping point calculated when AI cost < 50% of human cost. S-curve adoption modeled after tipping point. Aggregation: sub-tasks -> tasks -> jobs -> 24 BLS groups -> country. (Jitin walked through the full model)

- **Configurable simulation parameters:** capability doubling time (currently 7 months), tipping point threshold (currently 50% cost), adoption time 10-80% (currently 4 years), cost decline rate (currently 70%/year), adoption ceiling (currently 80%), replacement vs. productivity split (currently 50/50). (Jitin: "The model can act as a simulation. You can actually tune all of these parameters.")

- **Sub-task clustering** -- planned improvement to assign different parameters per cluster instead of uniform parameters across all 90,000 sub-tasks. (Jitin: "We are trying to cluster all of the subtasks into different categories and then for each cluster define a different set of parameters.")

- **Back-testing methodology** -- use first 3 months to identify ideal parameter values, test against next 3 months. Start with customer service and software engineering jobs. (Jitin proposed this approach)

- **Cost curve generalizability** -- the compliance framework enables analysis of new cost curves (e.g., DNA printing, GLP-1) even without prior analysis. (Tony: "When we have a new cost curve, say DNA printing, essentially, it would know how to do an analysis even though it hasn't done it before." Dr. Ali: "Correct. This is how we extrapolate to new cost curves.")

- **Domain Models** -- distinct from agents. Multiple specialized forecasting models per commodity/product within each domain (energy, metals, etc.).

- **Orchestration Layer** -- reformulator, clarification agent, lightweight agent, planning agent, main STDF agent, plus micro-agents being built.
